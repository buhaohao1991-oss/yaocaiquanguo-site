## 二、升级维度与具体方案

### 🎨 A. 前端 UI/UX 升级

#### A1. 首页工作台增强
**现状问题**：首页只是导航卡片，缺乏管理后台应有的数据概览。

**升级方案**：
- 顶部增加 4 个核心指标卡片（总基地数 / 溯源码已发 / 本月新增记录 / 链路完整率）
- 中部增加"最近动态"时间轴（从 activities 数据渠道生成）
- 底部保留模块快速入口，但改为更紧凑的列表+图标样式
- 增加"链路健康度"可视化（哪些基地还没关联种源/采收）

#### A2. 表单交互优化
**现状问题**：弹窗表单字段多，一次性铺开容易疲劳。

**升级方案**：
- 长表单改为分步（Step）模式，每步 3-5 个字段
- 增加必填项标记 `*` 和即时校验（失焦触发）
- 下拉选项增加搜索过滤（基地列表、种源列表等关联选择器）
- 日期选择器增加快捷按钮（今天 / 昨天 / 本周）

#### A3. 空状态与引导优化
**现状问题**："当前还没有记录"文案单调，新用户无引导路径。

**升级方案**：
- 空状态增加 SVG 插图（中药/基地/仓库主题）
- 首次进入增加引导 tooltip（"第一步：先建立基地档案"）
- 各模块空状态增加「推荐操作」按钮直接跳转

#### A4. 移动端适配强化
**现状问题**：侧边栏在小屏幕下不友好。

**升级方案**：
- 小屏幕下侧边栏改为抽屉式（点击汉堡菜单弹出）
- 表格在移动端改为卡片列表视图
- 表单弹窗改为全屏 sheet

#### A5. 视觉细节提升
- 侧边栏 logo 区域增加品牌动效
- 表格行增加悬停高亮和选中态动画
- 统计卡片增加数字滚动动画
- 加载状态增加骨架屏

### 🔧 B. 代码架构升级

#### B1. JS 模块化拆分
**现状问题**：3778 行单文件，维护困难。

**升级方案**（保持原生 JS，不引入构建工具）：
```
public/assets/
  trace-admin.js        → 主入口（路由分发 + 初始化，~200 行）
  modules/
    constants.js        → 常量、选项列表
    store.js            → localStorage 读写与数据转换
    views.js            → buildSharedData + 视图计算
    render-shell.js     → sidebar + app-shell 渲染
    render-home.js      → 首页渲染
    render-entity.js    → 通用实体页渲染
    render-compound.js  → 复合页渲染（种植+农事 / 加工+工艺）
    render-query.js     → 消费者溯源查询页
    dialogs.js          → 表单弹窗逻辑
    binding.js          → 事件绑定
    utils.js            → escapeHtml、formatNumber 等工具
```
使用 `<script type="module">` + ES import 实现，零构建工具。

#### B2. 数据层优化
**现状问题**：Base64 图片存 localStorage 容易撑满 5MB 配额。

**升级方案**：
- 图片压缩阈值降低（最大边 800px，quality 0.6）
- 增加 localStorage 用量监控（侧边栏底部显示占用百分比）
- 增加数据导出/导入功能（JSON 文件），防止数据丢失
- 预留 IndexedDB 升级路径（图片单独存 IndexedDB，元数据留 localStorage）

#### B3. 安全与健壮性
- 确认所有用户输入均通过 `escapeHtml` 处理（防 XSS）
- 增加 `try-catch` 包裹 JSON.parse 调用
- localStorage 读取增加版本迁移逻辑
- 增加全局错误边界（uncaught error → toast 提示）

### 🔗 C. 溯源链路跑通

#### C1. 全链路数据流
```
基地 → 种源（关联基地ID）→ 种植过程（关联基地+种源）→ 农事记录（挂在种植下）
  → 采收（关联种植）→ 加工过程（关联采收+药材）→ 工艺步骤（挂在加工下）
  → 溯源码（关联加工+仓库）→ 消费者扫码查询
```

#### C2. 链路完整性校验
- 每条溯源码生成时自动校验上游是否完整
- 缺失环节高亮标红（如：采收批次没有关联种植过程）
- 溯源查询页增加完整度评分（5星制）

#### C3. 消费者查询页增强
**现状问题**：查询页功能完整但视觉偏素。

**升级方案**：
- 增加时间轴动画（从基地→仓库逐步展开）
- 各环节增加照片展示（从各阶段记录中拉取）
- 增加地图定位（基地位置 → 仓库位置的连线）
- 底部增加企业信息和防伪声明

## 四、技术决策

**保持纯原生 JS/CSS，不迁移到 Vue/React。** 理由：
1. 当前 7500 行代码量完全在原生 JS 可控范围内
2. 纯静态部署是你的核心优势（零成本、零运维）
3. ES Module 拆分即可解决维护性问题
4. 避免引入构建工具增加复杂度

**不引入后端（当前阶段）。** 理由：
1. 溯源平台核心价值在于数据录入和展示，不在于多用户协作
2. localStorage + JSON 导出/导入已覆盖单用户场景
3. 后期如需多用户，可接入 Supabase / Firebase 等 BaaS
